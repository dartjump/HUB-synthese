Droits & Utilisateurs
Permissions Linux, propriétaires, groupes et gestion des utilisateurs
// Lire les permissions
🔒 Structure des permissions -rwxr-xr-- 1 liam groupe 1234 mai 10 script.sh - type (d=dossier, l=lien) · rwx propriétaire · r-x groupe · r-- autres · r=lecture, w=écriture, x=exécution
// chmod — Modifier les permissions
OctalMode numérique
r=4, w=2, x=1. On additionne par groupe
chmod 755 script.sh ← rwxr-xr-x chmod 644 fichier.txt ← rw-r--r-- chmod 600 secret.txt ← rw------- (propriétaire seul)
SymboliqueMode lettres
u=user, g=group, o=other, a=all
chmod +x script.sh ← ajouter exécution chmod u+w fichier.txt ← proprio peut écrire chmod o-r secret.txt ← autres ne lisent pas

Scripts Bash
Fichiers .sh — automatiser des séquences de commandes Linux
📄 DéfinitionUn script .sh est un fichier texte contenant des commandes Bash. La première ligne (shebang) indique l'interpréteur.
// Instructions de base
#!/bin/bashShebang — 1re ligne obligatoire
#!/bin/bash # Commentaire
#Commentaires
# Auteur : Liam # Date : 2025
echoAfficher un message
echo "Bonjour !" echo "" ← ligne vide
readSaisie utilisateur
read -p "Ton prénom : " prenom echo "Bonjour, $prenom !"
VariablesDéclarer et utiliser
Pas d'espaces autour du =
nom="Liam" echo "Bonjour $nom"
if / elseConditions
if [ "$nom" == "Liam" ]; then echo "Bonjour Liam !" else echo "Qui es-tu ?" fi
forBoucle for
for i in 1 2 3; do echo "Ligne $i" done
whileBoucle while
c=1 while [ $c -le 5 ]; do echo "Tour $c" c=$(( $c + 1 )) done
// Variables d'environnement
# Afficher toutes les variables : env # Variables prédéfinies importantes : echo $HOME ← /home/liam echo $USER ← liam echo $SHELL ← /bin/bash echo $PATH ← chemins des exécutables echo $PWD ← répertoire courant echo $? ← code retour de la dernière commande
// Exécuter un script
# 1. Créer le script nano monscript.sh # 2. Donner la permission d'exécution chmod +x monscript.sh # 3. Lancer ./monscript.sh # Ou sans permission : bash monscript.sh
