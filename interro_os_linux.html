04

Commandes avancées

# head / tail — premières et dernières lignes
head -3 essai.txt # 3 premières lignes
tail -3 essai.txt # 3 dernières lignes
head -3 essai.txt ; tail -3 essai.txt # les 2 ensemble
# grep — filtrer des lignes
grep "Lorem" essai.txt # lignes AVEC "Lorem"
grep -v "Lorem" essai.txt # lignes SANS "Lorem"
grep -i "lorem" essai.txt # insensible à la casse
# wc — compter
wc -l essai.txt # nombre de lignes
wc -c essai.txt # nombre de caractères
wc -w essai.txt # nombre de mots
# sed — substitution
sed 's/chat/chien/' f.txt # remplace 1ère occurrence par ligne
sed 's/chat/chien/g' f.txt # remplace TOUTES les occurrences
05

Redirections & Pipes

Flux standards
  • stdin (0) — entrée = clavier
  • stdout (1) — sortie normale = terminal
  • stderr (2) — sortie d'erreur = terminal
À retenir pour l'interro !
  • > écrase le fichier
  • >> ajoute à la fin
  • | chaîne deux commandes
  • 2> redirige les erreurs
  • && si 1ère réussit
  • ; dans tous les cas
  • || si 1ère échoue
>Redirige stdout vers un fichier (écrase)ls > liste.txt
>>Redirige stdout (ajoute à la fin)ls >> liste.txt
2>Redirige stderr vers un fichiercmd 2> erreurs.txt
<Redirige stdin depuis un fichiersort < liste.txt
|Pipe — envoie stdout → stdin de la commande suivantels | grep ".txt"
&&Exécute cmd2 seulement si cmd1 réussit (code retour 0)apt update && apt upgrade
;Exécute cmd2 dans tous les cashead -3 f ; tail -3 f
||Exécute cmd2 seulement si cmd1 échouecmd1 || echo "raté"
# Exemples pratiques pipes
cat essai.txt | grep "Lorem" | wc -l # compter les lignes avec "Lorem"
ls | sort # liste triée
ls inexistant 2> /dev/null # supprimer les erreurs

755 = rwxr-xr-x
644 = rw-r--r-- (fichier normal)

# chmod — modifier les droits
chmod 755 fichier # numérique
chmod u+x script.sh # ajouter execute au proprio
chmod go-w fichier # enlever write groupe et autres
chmod -R 755 dossier/ # récursif
# chown — changer le propriétaire
sudo chown bob fichier.txt # changer proprio
sudo chown bob:bob fichier.txt # proprio + groupe
# chgrp — changer le groupe propriétaire
sudo chgrp technicien fichier.txt # changer le groupe
# passwd — changer mot de passe
passwd bob # changer le mot de passe de bob
u g o a
  • u = user (proprio)
  • g = group
  • o = other
  • a = all (tous)
+ - =
  • + = ajouter le droit
  • - = supprimer le droit
  • = = affecter exactement
Fichiers clés
  • /etc/passwd — users
  • /etc/shadow — mots de passe
  • /etc/group — groupes
07

Gestion des utilisateurs & groupes

CommandeSyntaxeDescriptionExemple
adduser nom Crée un utilisateur + répertoire home (interactif, Debian/Ubuntu) sudo adduser bob
deluser nom Supprime un utilisateur (Debian/Ubuntu) sudo deluser bob
addgroup nom Crée un groupe (Debian/Ubuntu) = groupadd sudo addgroup technicien
delgroup nom Supprime un groupe (Debian/Ubuntu) = groupdel sudo delgroup technicien
usermod -aG -g -l Modifie un utilisateur. -aG=ajouter à un groupe, -g=groupe principal, -l=renommer sudo usermod -aG bob technicien
passwd nom Changer le mot de passe d'un utilisateur sudo passwd bob
groups nom Affiche les groupes auxquels appartient un utilisateur groups bob
⚠️
adduser vs useradd : Sur Ubuntu/Debian, adduser est plus pratique (crée home, demande mdp interactivement). useradd est universel mais basique. Pareil pour addgroup / groupadd.
08

Wildcards (masques)

*Remplace 0 ou plusieurs caractères quelconquesls *.txt
?Remplace exactement 1 caractère quelconquels file?.txt
[]Un caractère parmi un ensemble donnéls file[123].txt
[!]Un caractère sauf ceux listésls [!A-Z]*
# Exemples du TP03
cat e* # fichiers dont le nom commence par "e"
cat *s* # fichiers contenant au moins 1 "s"
cat *.txt # fichiers se terminant par ".txt"
cat e*s*.txt # commence par e + contient s + .txt
cat ??s* # "s" en 3ème caractère (2 chars avant)
cp *.txt DIR1/ # copier tous les .txt dans DIR1
rm ????.txt # exactement 4 chars + .txt
ls [A-Z]* # commence par une majuscule
ls *.[ch] # fichiers .c ou .h
09

Arborescence Linux

/← racine de tout le système
├── bin/commandes essentielles (ls, cp, mv, rm…)
├── sbin/commandes système (root seulement)
├── etc/fichiers de configuration (/etc/passwd, /etc/group…)
├── home/répertoires des utilisateurs
│   └── liam/← c'est ton ~ (home)
├── root/home du super-admin root
├── usr/programmes installés (apt…)
├── var/données variables (logs…)
├── tmp/fichiers temporaires — sticky bit actif
├── dev/périphériques (disques, terminaux…)
│   └── null← poubelle : 2> /dev/null
├── proc/informations sur les processus (virtuel)
└── lib/bibliothèques partagées
