01 Le Terminal Linux
Démarrage, structure du prompt et raccourcis essentiels

// Ouvrir un terminal

Méthode 1Raccourci clavier
Sur la plupart des distributions Ubuntu/Debian
Ctrl + Alt + T
Méthode 2Menu Applications
Chercher "Terminal", "Konsole", "xterm" ou "GNOME Terminal" dans le menu
Applications → Outils système → Terminal
RootMode Super-utilisateur
Passer en root pour les commandes d'administration (avec précaution !)
sudo -i ← shell root interactif sudo commande ← exécuter en root su - ← changer d'utilisateur root

// Structure du prompt

📍 Lire le prompt Le prompt Linux contient des informations essentielles. Par exemple : liam@ubuntu:~/Documents$
  • liam — nom de l'utilisateur connecté
  • @ubuntu — nom de la machine (hostname)
  • :~/Documents — répertoire courant (~ = home)
  • $ — utilisateur normal (# = root)

Suspendre un processus (mettre en arrière-plan)

💡 Bash vs sh Bash (Bourne Again SHell) est le shell par défaut sur la plupart des distributions Linux. Il est plus puissant que sh (Bourne Shell) et supporte les tableaux, l'historique interactif, et l'autocomplétion. La commande echo $SHELL affiche le shell actuel.

// Répertoires

mkdirMake Directory — Créer un dossier
Crée un ou plusieurs répertoires. -p crée les parents si besoin.
mkdir MonDossier mkdir -p /home/liam/T1012/Labo2 Créer plusieurs d'un coup : mkdir -p projet/{src,docs,tests}

Chemins absolus (depuis /) ou relatifs (depuis ici)

04 Droits & Utilisateurs
Permissions Linux, propriétaires, groupes et gestion des utilisateurs

// Lire les permissions

🔒 Structure des permissions -rwxr-xr-- 1 liam groupe 1234 mai 10 script.sh
  • - → type : -=fichier, d=dossier, l=lien
  • rwx → propriétaire : lecture (r), écriture (w), exécution (x)
  • r-x → groupe : lecture + exécution, pas d'écriture
  • r-- → autres : lecture seulement

// chmod — Modifier les permissions

Notation octalMode numérique
r=4, w=2, x=1. On additionne par groupe (proprio / groupe / autres)
chmod 755 script.sh ← rwxr-xr-x chmod 644 rapport.txt ← rw-r--r-- chmod 777 public/ ← tout le monde tout chmod 600 secret.txt ← proprio seul rw
Notation symboliqueMode lettres
u=user, g=group, o=other, a=all
chmod +x script.sh ← ajouter exécution chmod u+w fichier.txt ← proprio peut écrire chmod o-r secret.txt ← autres ne lisent pas chmod a+r public.txt ← tout le monde lit

06 Scripts Bash
Fichiers .sh — automatiser des séquences de commandes Linux

// Qu'est-ce qu'un script bash ?

📄 Définition Un script .sh est un fichier texte contenant une suite de commandes Bash qui s'exécutent automatiquement dans l'ordre. La première ligne (shebang) indique quel interpréteur utiliser.

// Instructions de base

#!/bin/bashShebang — première ligne obligatoire
Indique que le script doit être exécuté par bash
#!/bin/bash # Les commandes suivent ensuite
#Commentaires
Les lignes commençant par # sont des commentaires
# Ceci est un commentaire # Auteur : Liam # Date : 2025
echoAfficher un message
Afficher du texte dans le terminal
echo "Bonjour !" echo "" ← ligne vide echo "Fin du script"
readLire une entrée utilisateur
Demander une valeur à l'utilisateur
read -p "Ton prénom : " prenom echo "Bonjour, $prenom !"
VariablesDéclarer et utiliser
Pas d'espaces autour du =. Utiliser $ pour accéder.
nom="Liam" age=20 echo "Je m'appelle $nom, j'ai $age ans"
if / elseConditions
Structure conditionnelle en bash
if [ "$nom" == "Liam" ]; then echo "Bonjour Liam !" else echo "Qui es-tu ?" fi
forBoucle for
Itérer sur une liste de valeurs
for i in 1 2 3 4 5; do echo "Ligne $i" done for f in *.txt; do echo "Fichier : $f" done
whileBoucle while
Répéter tant qu'une condition est vraie
compteur=1 while [ $compteur -le 5 ]; do echo "Tour $compteur" compteur=$(( $compteur + 1 )) done

// Variables d'environnement

Les variables d'environnement sont des variables prédéfinies par le système. On y accède avec $.
# Afficher toutes les variables : env printenv # Variables prédéfinies importantes : echo $HOME ← /home/liam echo $USER ← liam echo $SHELL ← /bin/bash echo $PATH ← chemins des exécutables echo $HOSTNAME ← nom de la machine echo $PWD ← répertoire courant echo $? ← code retour de la dernière commande

// Exécuter un script

🚀 Méthode Un script doit avoir la permission d'exécution pour pouvoir être lancé directement.
# 1. Créer le script nano monscript.sh # 2. Lui donner la permission d'exécution chmod +x monscript.sh # 3. L'exécuter ./monscript.sh # Ou sans permission : bash monscript.sh

// Exemple de script complet

monscript.sh
#!/bin/bash # MonScript.sh — Script de démonstration clear echo "========================================" echo " Informations réseau de la machine" echo "========================================" echo "" # Afficher l'utilisateur et la machine echo "Utilisateur : $USER" echo "Machine : $HOSTNAME" echo "Répertoire : $PWD" echo "" # Infos réseau ip a | grep "inet " echo "" read -p "Appuyez sur Entrée pour continuer..."

// Démarrage automatique

🚀 Méthode Pour exécuter un script automatiquement à chaque ouverture de terminal, ajouter la commande dans ~/.bashrc (ou ~/.bash_profile pour les connexions). Pour un démarrage système, utiliser cron ou systemd.
# Ajouter à la fin de ~/.bashrc : nano ~/.bashrc # Puis ajouter la ligne : bash ~/monscript.sh # Recharger sans fermer le terminal : source ~/.bashrc
